# Z-image 新手部署步骤

这份文件是专门给你现在这个仓库写的：

- GitHub 账号：`tommykid13`
- 仓库：`https://github.com/tommykid13/Z-image`
- 本地目录：`D:\GitHub\Z-image`

如果你是 0 基础，先只做下面这 6 步。

## 第 1 步：先确认项目已经在本地

打开 `D:\GitHub\Z-image`，你应该能看到这些文件：

- `package.json`
- `src`
- `api`
- `data`
- `.env.example`

如果这些文件都在，说明项目代码已经放好了。

## 第 2 步：先在本地跑起来

打开 PowerShell，执行：

```powershell
cd D:\GitHub\Z-image
npm install
npm run dev
```

看到本地地址后，通常打开：

- `http://localhost:5173`

这一步先不用登录，也不用配 GitHub Token。页面应该能先打开。

## 第 3 步：创建本地 `.env` 文件

在 `D:\GitHub\Z-image` 目录里，新建一个文件：

- `.env`

把下面内容复制进去：

```env
SESSION_SECRET=这里先随便填一个很长的随机字符串
GITHUB_OAUTH_CLIENT_ID=这里先留空，等会儿去 GitHub 复制
GITHUB_OAUTH_CLIENT_SECRET=这里先留空，等会儿去 GitHub 复制
GITHUB_OAUTH_REDIRECT_URI=http://localhost:5173/api/auth/github/callback
GITHUB_ADMIN_USERS=tommykid13
GITHUB_OWNER=tommykid13
GITHUB_REPO=Z-image
GITHUB_BRANCH=main
GITHUB_TOKEN=这里先留空，等会儿去 GitHub 复制
DATA_PATH=data/prompts.json
UPLOAD_DIR=public/uploads
```

注意：

- `.env` 不要发给别人
- `.env` 不要截图公开
- 里面将来会放你的密钥

## 第 4 步：创建 GitHub OAuth App

你需要去 GitHub 网页操作。

打开：

- [GitHub Developer Settings](https://github.com/settings/developers)

然后：

1. 点击 `OAuth Apps`
2. 点击 `New OAuth App`
3. 按下面填写

建议填写：

- `Application name`: `Z-image Admin`
- `Homepage URL`: `http://localhost:5173`
- `Authorization callback URL`: `http://localhost:5173/api/auth/github/callback`

创建完成后你会得到：

- `Client ID`

然后点击：

- `Generate a new client secret`

你还会得到：

- `Client Secret`

把它们填回 `.env`：

```env
GITHUB_OAUTH_CLIENT_ID=你刚刚复制的Client ID
GITHUB_OAUTH_CLIENT_SECRET=你刚刚复制的Client Secret
```

## 第 5 步：创建 GitHub Token

打开：

- [GitHub Fine-grained Tokens](https://github.com/settings/personal-access-tokens)

然后：

1. 点击 `Generate new token`
2. 选择你的仓库 `Z-image`
3. Repository permissions 里把 `Contents` 设为 `Read and write`

生成后复制 token，填进 `.env`：

```env
GITHUB_TOKEN=你刚刚生成的token
```

## 第 6 步：重新启动项目并测试登录

回到 PowerShell：

```powershell
cd D:\GitHub\Z-image
npm run dev
```

然后在网页上：

1. 打开首页
2. 点击右上角 `GitHub 管理员登录`
3. 用你的 GitHub 账号 `tommykid13` 登录
4. 登录成功后，打开后台
5. 试着改一个 Prompt
6. 点击“保存到 GitHub”

如果成功，说明：

- OAuth 正常
- 管理员白名单正常
- GitHub 写回正常

## 第 7 步：把代码推到 GitHub

如果本地测试没问题，再执行：

```powershell
cd D:\GitHub\Z-image
git add .
git commit -m "feat: init z-image prompt manager"
git push -u origin main
```

## 第 8 步：部署到 Vercel

登录 Vercel 后导入仓库：

- `tommykid13/Z-image`

然后在 Vercel 里把 `.env` 里的那些变量，也一模一样填到：

- `Project Settings`
- `Environment Variables`

生产环境最重要的是把回调地址改成你线上域名：

```env
GITHUB_OAUTH_REDIRECT_URI=https://你的-vercel-域名/api/auth/github/callback
```

同时 GitHub OAuth App 里的回调地址，也要改成同一个线上地址。

## 现在最推荐你先做什么

如果你只想先把第一步跑通，先做这两个：

1. `npm install`
2. `npm run dev`

等你本地页面先打开了，我们再继续配 OAuth 和 Token，会轻松很多。
